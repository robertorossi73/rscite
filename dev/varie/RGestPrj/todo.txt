------------------------------ Gestore Progetto -------------------------------

Questo gestore progetti gestisce i file con estensione .rsp.

Questa tipologia di file ha lo scopo di racchiudere le impostazioni necessarie ad alcune funzioni, come ad esempio un'eventuale gestore ftp (nome utente, server, porta, ecc...), o una ricerca di un termine in tutti i file che compongono un progetto (es. quali tipi di file ne fanno parte, la cartella home del progetto), oppure l'esecuzione di un gestore di revisioni come GIT.

Ecco l'idea.

Immaginiamo di avere il file "test.php", facente parte di un progetto più ampio, ed immaginiamo sia in questo percorso :

  c:\sviluppo\sito_web\dir_a\dir_b\test.php
  
A questo punto a RSciTE servirà, in locale, un posto nel quale leggere i dati relativi al progetto che racchiude il nostro file.

Immaginiamo di creare un file di configurazione per i dati necessari, e supponiamo che venga posizionato nella cartella principale del nostro progetto "c:\sito_web\", infine chiamiamo il file ci configurazione "RSciTE.rsp" (che risiederà quindi in "c:\sito_web\RSciTE.rsp").

Quando si ordinerà a RSciTE di eseguire, ad esempio la ricerca di un simbolo in tutto il progetto, il programma risalità la gerarchia di cartelle fino a trovare il file RSciTE.rsp, trovato il quale lo utilizzerà per ricavare i dati necessari.

Nel nostro caso, RSciTE risalità la gerarchia data dal percorso "c:\sviluppo\sito_web\dir_a\dir_b\" fino ad incontrare un file di progetto (RSciTE.rsp).

Il file di progetto potrà contenere, inizialmente, questi dati :

* Nome del progetto;
* percorso cartella principale/home del progetto. Questo percorso è opzionale, senza il quale il percorso base sarà quello dove risiede il file di progetto stesso;

In futuro :
* dati per accesso a eventuale server FTP;
* dati per accesso a GIT
* dati per effettuare backup intero progetto;

------------------------- Le funzioni da implementare -------------------------

Le funzioni previste dal nuovo modulo di gestione progetti :

* Apertura progetto relativo a file corrente;
* Crea nuovo file di progetto;
* Cerca simbolo in progetto (con l'uso di CTAGS). Questa funzione sarebbe meglio inserirla nella procedura di gestione di CTAGS;
* Genera Elenco file presenti nel progetto (usando DIR). Questa funzione chiederà le estensioni da elencare, ricordando l'ultima scelta fatta;

Funzioni previste per il futuro :

* Informazioni statistiche sul progetto (come numero linee di codice, linee di commento, dimensione file del progetto, ecc...)
* Capacità di utilizzare un sistema di controllo delle versioni (inizialmente GIT), sarebbe sufficiente poter lanciare l'interfaccia grafica di git sulla cartella del progetto.
